Use as a library¶

Quevedo can be used as a command line application to manage a dataset, but it can also be used from other Python code to make programatic access to the dataset more convenient. The documentation for the main objects that might be useful is included below.

Example¶

from quevedo import Dataset, Target

ds = Dataset('path/to/the/dataset')

ds.get_single(Target.GRAPH, 'subset', 32)

Dataset¶

class
quevedo.dataset.Dataset(path)

Class representing a Quevedo dataset.

It provides access to the annotations, subsets, and any neural networks contained.

Parameters

Methods
  • create() — Create or initialize a directory to be a Quevedo dataset.
  • create_subset(target, name, existing) — Creates the directory for a new subset. If it exists, behaviour is controlled by existing. It can be 'a' to abort (the default), 'r' to remove existing annotations, or 'm' (merge) to do nothing. Returns the creeated path.
  • get_annotations(target, subset) — Returns a generator that yields all annotations, those of a given target, or only those in a given subset (or subsets) and target.
  • get_subsets(target) — Gets information about the subsets in a given target.
  • list_networks() — Get a list of all neural networks for this dataset.

Target¶

Quevedo annotations are of two types, logograms and graphemes, both derived from the parent class Annotation. When it is necessary to distinguish logograms and graphemes in a process, there is the enum Target, which can take the values Target.GRAPH or Target.LOGO. the values Target.GRAPH or Target.LOGO.

Network¶

class
quevedo.network.network.Network(dataset, name)

Class representing a neural net to train and predict logograms or graphemes.

Methods
  • auto_annotate(annotation) — Use the network to annotate a real instance, possibly already partially annotated.
  • filter(annotation) — Override to control the annotations included in training by checking their tags.
  • get_net_config(num_classes) — Get the darknet architecture (.cfg file contents) for this net.
  • is_prepared() — Checks whether the neural network configuration files have been made.
  • is_trained() — Checks whether the neural network has been trained and can be used to predict.
  • load() — Loads the weights for the trained neural network so it can be used to predict.
  • predict(image_path) — Use the trained neural network to predict results from an image.
  • prepare() — Creates the files needed for training and testing darknet.
  • prepare_annotation(annotation, num, tag_set) — Prepare the files needed to train this annotation. Return the name that the final file (link or copy) should have.
  • test(annotation, stats) — Use the network to get the prediction for a real annotation, compare results and update stats. See test.py for stats.
  • train(initial) — Trains the neural network. When finished, removes partial weights and keeps only the last. Can be interrupted and optionally resumed later.
  • update_tag_set(tag_set, annotation) — Add the relevant tag from this annotation to the tag set (used while collecting all tags prior to training).
